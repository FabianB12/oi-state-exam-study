Module 1

PAC, online learning, and VC dimension

PAC asks whether random labeled examples are enough to output a low-error hypothesis. Online learning asks whether sequential feedback can keep mistakes or regret small, even when examples are adversarial.

Committee priorityNormal focus

Useful SMU foundation, but committee overlap is weaker than the decision-making and reinforcement-learning material.

  • PAC guarantee, epsilon/delta, and VC dimension
  • Online mistake bounds and regret framing
  • PAC vs adversarial online assumptions

Say this out loud

What you must be able to say

  • PAC uses i.i.d. examples from an unknown distribution and wants true error at most epsilon with probability at least 1-delta.
  • Efficient PAC learning also requires polynomial running time and polynomial sample complexity.
  • Online learning predicts before seeing the label; performance is a mistake bound or regret against a comparator.
  • Finite mistake-bound online learning implies realizable PAC learnability, but PAC does not guarantee low adversarial online mistakes.
  • VC dimension is the largest shattered set size and characterizes distribution-free binary PAC learnability.
  • Finite VC dimension is the official PAC characterization; Littlestone dimension is a useful optional one-sentence online analogue for realizable adversarial learning.

Remember: PAC is about random training samples; online learning is about a sequence where each prediction happens before feedback.

Module 2

Conjunctions, disjunctions, and reductions

The Boolean learner is the cleanest part of SMU: start too strict, remove literals that positive examples disprove, and use duality for disjunctions.

45-60 min

Watch first

PAC learning - Georgia Tech ML (8:03)

Fallback link

Optional text backup: mistake-bound learning slides

This subtopic is under-served by short videos. Use the lab plus the formal notes, and keep the linked lecture slides as backup for the online mistake-bound framing.

descriptionOpen slides

Say this out loud

What you must be able to say

  • A conjunction over n Boolean variables has literals x_i and not x_i.
  • Start with all 2n literals; for every positive example, remove every literal falsified by that example; ignore negative examples.
  • The algorithm is O(mn), because each example can check all literals.
  • Disjunctions use the dual: remove literals made true by negative examples, or reduce using De Morgan.
  • Online conjunction learning removes a literal after a false negative; each literal disappears at most once, so the update bound is at most 2n.
  • Online disjunction learning is the dual: false positives delete literals made true by the current negative example, again with at most one deletion per literal.
  • k-CNF becomes conjunction learning over all clauses of size at most k, which is efficient for fixed k because there are O(n^k) clauses.

Remember: for conjunctions, positive examples delete impossible literals; for disjunctions, swap the logic to negative examples.

Module 3

Multi-armed bandits

Bandits are one-step decision problems with partial feedback. You only see the reward of the arm you pulled, so exploration is the cost of learning what the other arms might have done.

Committee prioritySolid focus

Partial-feedback decision making is a plausible bridge into planning/RL-style questioning even when SMU is not the strongest committee signal.

  • Expected regret and gap decomposition
  • Epsilon-greedy vs UCB vs Thompson sampling
  • Exploration-exploitation tradeoff
65-85 min

Fallback link

UCB and Thompson sampling comparison/detail (9:07 + 13:16)

Fallback link

Say this out loud

What you must be able to say

  • At each round, choose one of K arms and observe only that arm's reward.
  • Expected regret is T times the best mean minus expected collected reward, equivalently sum of gaps times expected pulls of suboptimal arms.
  • Fixed epsilon-greedy keeps exploring forever, so regret is linear; decaying epsilon is schedule-sensitive.
  • UCB chooses empirical mean plus confidence bonus sqrt(2 ln t / N_i(t)), giving optimism under uncertainty.
  • UCB1 has logarithmic regret for bounded stochastic rewards.
  • Thompson sampling samples from each posterior and pulls the largest sample; Bernoulli arms use Beta alpha/beta success/failure updates.

Remember: regret measures what you lost by exploring or choosing weak arms instead of the best arm in hindsight.

Remember: epsilon explores randomly, UCB explores uncertain arms, and Thompson explores by sampling a plausible world.

Module 4

MDPs, values, Bellman equations, and value iteration

An MDP adds state and delayed reward. Bellman equations are just "reward now plus discounted value later", optimized over actions.

Committee priorityHeavy focus

Decision processes, Bellman backups, and value iteration line up with committee interest in planning, games, and sequential decision making.

  • MDP tuple, policies, V and Q functions
  • Bellman optimality equation
  • Value-iteration backup and greedy policy extraction
60-75 min

Watch first

MDPs, values, policies, and value iteration (83:20 + 16:39)

Fallback link

Say this out loud

What you must be able to say

  • An MDP is (S, A, P, R, gamma): states, actions, transition probabilities, rewards, and discount factor.
  • A policy pi(a|s) defines behavior; V^pi(s) values a state, Q^pi(s,a) values taking an action then following pi.
  • The optimal Bellman equation maximizes expected immediate reward plus gamma times next-state optimal value.
  • Value iteration repeatedly applies the Bellman optimality backup until values stabilize, then extracts a greedy policy.
  • For finite discounted MDPs, value iteration converges.

Remember: V asks how good a state is; Q asks how good one action is from that state.

Say this out loud

What you must be able to say

  • Direct utility estimation is Monte Carlo prediction: average sampled returns G_t for each state.
  • ADP is model-based: estimate P and R from counts/rewards, then solve the learned MDP with value or policy iteration.
  • TD updates V(s) toward r + gamma V(s'), mixing sampling with bootstrapping.
  • Q-learning is off-policy: target r + gamma max_a' Q(s',a').
  • SARSA is on-policy: target r + gamma Q(s',a') where a' is the actually selected next action.
  • Exploration strategies include epsilon-greedy, softmax, optimistic initialization, UCB bonuses, and posterior sampling.
  • Policy search directly optimizes a parameterized policy objective J(theta); know the concept, with policy gradients/REINFORCE/actor-critic as optional examples.

Remember: Q-learning uses the best next action in the target; SARSA uses the action the current policy actually took.

MDP tuple and Bellman computation

Committee priorityHeavy focus

Reported overlap plus Bellman-style computation makes this the highest-yield SMU drill despite lower report confidence.

  • MDP tuple and V/Q definitions
  • Bellman/value-iteration computation
  • Worked transition-and-reward backup

Context: OI AI report, lower-confidence overlap: MDPs, definition, computation, and a partially specified worked example.

Show model answer
  • An MDP is (S,A,P,R,gamma): states, actions, transition probabilities P(s'|s,a), rewards, and discount factor.
  • V^pi(s) is expected discounted return from state s under policy pi; Q^pi(s,a) is expected return after taking action a and then following pi.
  • For value iteration, compute each action score as sum_s' P(s'|s,a)(R(s,a,s') + gamma V_k(s')), then set V_{k+1}(s) to the maximum score.
  • In a worked example, plug each listed successor probability and reward into the sum, add the discounted old successor value, multiply by the probability, sum the terms, and compare actions.
  • After convergence, choose the greedy action with the highest backed-up score in each state.
