Complexity & graph representation

Before any graph algorithm, you need two instincts: how the input size grows, and what representation makes the operations cheap. Not glamorous, but it prevents almost every later complexity mistake.

Committee priorityNormal focus

Useful common-core support for PAL answers, especially when reports ask for implementation memory and runtime tradeoffs.

  • asymptotic notation and input-size reasoning
  • adjacency matrix, list, and incidence matrix tradeoffs
  • sparse versus dense graph algorithm choices

Say this out loud

Must be able to say

  • Big-O is an upper asymptotic bound: f(n) ∈ O(g(n)); Omega is lower: f(n) ∈ Ω(g(n)); Theta is tight: f(n) ∈ Θ(g(n)).
  • For graph algorithms, use n = |V| and m = |E|.
  • Adjacency matrix: O(n^2) memory, O(1) edge lookup, good for dense graphs.
  • Adjacency list: O(n + m) memory, good for sparse traversal.
  • Distance matrix stores all-pairs shortest-path distances; it is often an output, not just an input representation.
  • Laplacian is degree matrix minus adjacency matrix: L = D - A. The diagonal stores vertex degrees, and neighbor links become -1.

Notation visual

Big-O: upper bound

f(n) ∈ O(g(n)) means there are constants c > 0 and n0 such that 0 ≤ f(n) ≤ c g(n) for every n ≥ n0.

Omega: lower bound

f(n) ∈ Ω(g(n)) means there are constants c > 0 and n0 such that 0 ≤ c g(n) ≤ f(n) for every n ≥ n0.

Theta: tight bound

f(n) ∈ Θ(g(n)) means there are constants c1,c2 > 0 and n0 such that 0 ≤ c1 g(n) ≤ f(n) ≤ c2 g(n) for every n ≥ n0.

Remember: Big-O is a ceiling, Omega is a floor, Theta means the function is squeezed between both after some n0.

Lab

How a graph becomes arrays

Use this tiny undirected graph: A-B, A-C, B-D. The important exam distinction is: an adjacency list is usually an array indexed by vertex, where each cell stores that vertex's neighbors.

Remember: the outer array chooses the vertex; the inner array/list stores only that vertex's outgoing neighbors.

1. Edge list: one array of edges

edges = [(A,B), (A,C), (B,D)]

Yes, this is a single array. It is simple, but finding all neighbors of A means scanning the whole edge array.

2. Adjacency list: array of neighbor arrays

0 / A→B, C
1 / B→A, D
2 / C→A
3 / D→B

This is what people usually mean by "adjacency list". In code it is often vector<vector<int>> adj or List<Integer>[] adj.

3. Compact form: offsets + one neighbor array

neighbors = [B, C, A, D, A, B]
offsets   = [0, 2, 4, 5, 6]

Neighbors of vertex i are in neighbors[offsets[i] ... offsets[i+1]-1]. So A uses positions 0..1: B, C. This is a real "single big array" version, but it needs the offset array too.

B-tree insertion oral drill

  • Search down by separator keys until the target leaf.
  • Insert the key in sorted order; if the node overflows, split around the median separator.

Clearly relevant

Automata for text search

Committee prioritySolid focus

Direct report evidence covers automata, damaged words, dictionary automata, and multiple-pattern search.

  • KMP-style automaton search
  • approximate matching with Hamming or Levenshtein distance
  • Aho-Corasick for many patterns

Context: OI HCI report asked about automata, text search, damaged words, dictionary automata, and searching multiple words at once.

Show model answer
  • Exact one-pattern search can be viewed as a DFA over prefix lengths; KMP implements the same failure-transition idea in O(n + m).
  • A DFA has one active state; an NFA has a set of active states and can be simulated with bitsets or determinized by subset construction.
  • Damaged words mean approximate matching: Hamming distance allows substitutions only; Levenshtein allows insert, delete, substitute and is handled by DP or bounded-error automata.
  • Dictionary matching uses a trie plus failure links: Aho-Corasick scans once and reports all matched words in O(n + z) after construction.
  • Multiple words can be searched at once by the dictionary automaton; outputs attached to states report which patterns ended there.
Clearly relevant

B-tree insertion

Committee priorityMaximum focus

Directly matches the reported B-tree/B+ tree indexing prompt and the Bača-weighted tree emphasis.

  • B-tree definition and height intuition
  • find, insert, split, and promoted median
  • B+ tree leaf records and range scans

Context: OI Mgr Software Engineering common question asked to define B-trees, demonstrate find and insertion, compare B+ trees, and name other search trees.

Show model answer
  • A B-tree is a balanced multiway search tree optimized for disk/pages: each node stores several sorted keys and children, and all leaves are at the same depth.
  • Find chooses the interval between keys inside a node and descends to the matching child; height is O(log_t n) for minimum degree t.
  • Insert into the correct leaf; if a node overflows, split it and promote the median key upward, possibly cascading to a new root.
  • A B+ tree keeps records, or all keys with payload pointers, in linked leaves; internal nodes are separators, so range scans are especially efficient.
  • Other search trees: AVL and red-black trees keep binary height logarithmic; splay trees are amortized; skip lists use randomized levels.
Clearly relevant

SCC algorithms

Committee priorityHeavy focus

Direct report evidence asks for SCC definitions, Kosaraju-Sharir, Tarjan, correctness, and complexity.

  • maximal mutual reachability and condensation DAG
  • Kosaraju-Sharir finish order and reversed graph
  • Tarjan lowlink, stack, and linear time

Context: MSZZ OI Data Science report asked about strongly connected components, Kosaraju-Sharir, Tarjan, correctness, and time complexity.

Show model answer
  • An SCC is a maximal set of vertices mutually reachable in a directed graph; contracting SCCs gives an acyclic condensation graph.
  • Kosaraju-Sharir: run DFS, order vertices by decreasing finish time, then DFS the reversed graph in that order; each second-pass tree is one SCC.
  • Correctness uses the condensation DAG: finish-time order chooses a source/sink component in the reversed pass so DFS cannot leak into an unassigned different SCC.
  • Tarjan uses one DFS with discovery indices, lowlink values, and a stack of active vertices; when lowlink[v] = index[v], pop one SCC.
  • Both run in O(n + m); Kosaraju uses two passes and the reverse graph, Tarjan uses one pass plus stack bookkeeping.
Lower confidence

MST definitions and algorithms

Committee prioritySolid focus

Reported MST prompts make this a high-yield graph-algorithm rehearsal, even with lower confidence on the source.

  • tree equivalences and MST definition
  • Kruskal with union-find and Boruvka phases
  • cut property and runtime notes

Context: Bachelor-looking LGR report asked for tree definitions, cycles, MST, Kruskal/Boruvka, implementation, memory, and time complexity.

Show model answer
  • For an undirected graph, a tree is connected and acyclic; equivalently it has n - 1 edges and a unique simple path between every pair of vertices.
  • An MST is a minimum-weight spanning tree of a connected weighted undirected graph.
  • Kruskal sorts edges by weight and adds an edge if it connects two different components; union-find detects cycles. Time is dominated by sorting: O(m log m).
  • Boruvka repeatedly lets every component choose its lightest outgoing edge and contracts chosen edges; it finishes in O(log n) phases and about O(m log n) time.
  • Correctness rests on the cut property: the lightest edge crossing a cut is safe for some MST.
Lower confidence

Directed components

Reported priorityNormal focus

Lower-confidence report, but it targets the same graph vocabulary as SCC questions: directions, weak components, strong components, and condensation.

  • weak vs strong connectivity
  • mutual reachability
  • condensation graph intuition

Context: Bachelor-looking LGR report asked about directed graphs, connected components versus SCCs, examples, and rooted trees.

Show model answer
  • A directed graph has oriented edges; reachability must respect edge directions.
  • Weak connected components ignore edge directions and compute ordinary components of the underlying undirected graph.
  • SCCs require mutual directed reachability and are maximal under that property; every SCC lies inside one weak component.
  • A directed cycle can make one SCC; a directed rooted tree with edges away from the root usually has singleton SCCs but one weak component.
  • The condensation graph of SCCs is a DAG and is the right object for reasoning about directed-component structure.

Rehearsal

10-15 minute oral skeleton

I describe graph algorithms using asymptotic notation with n = |V| and m = |E|. Representation matters: adjacency matrices are good for dense graphs and constant edge queries, while adjacency lists are compact for sparse traversal.

For MSTs, Prim grows one tree, Kruskal grows a forest using union-find, and Boruvka merges components in phases. Correctness comes from the cut and cycle properties.

For directed graphs, SCCs are maximal mutually reachable sets. Kosaraju uses two DFS passes and reversed edges, while Tarjan uses one DFS with lowlink values. Euler trails are decided by degree balance conditions and constructed by Hierholzer.

Generation algorithms are governed by output size: 2^n subsets, C(n,k) k-subsets, and n! permutations. The sieve marks multiples for primes, and LCGs generate deterministic pseudorandom sequences.

Search trees fight height: AVL and red-black trees keep BSTs logarithmic, B/B+ trees are for pages and range queries, splay trees are amortized, k-d trees are spatial, and skip lists use random levels.

Automata connect to text search: regex, NFA, and DFA describe regular languages. KMP and Boyer-Moore are exact matching algorithms; edit distance handles approximate matching; Aho-Corasick handles dictionary matching.
