Module 1

ILP, formulations & branch and bound

ILP is the modeling language of discrete decisions. Learn the four patterns: yes/no variables, assignment variables, flow conservation, and ordering variables. Branch and bound is the exact search wrapper when the formulation is hard.

Committee priorityHeavy focus

Komenda/CSP overlap and Bureš-style constrained testing both reward clean formulations and exact-search reasoning.

  • binary, assignment, flow, and ordering variables
  • branch and bound bounds, incumbents, and pruning
  • TSP and shortest-path ILP formulations
60-75 min

Say this out loud

What you must be able to say

  • LP optimizes a linear objective over linear constraints; ILP adds integer or binary variables and is NP-hard in general.
  • Shortest path ILP uses edge variables and flow conservation; the relaxation is integral for the network matrix.
  • TSP ILP needs degree constraints and subtour elimination constraints.
  • Typical official formulations use the same patterns: binary choice variables, assignment variables, conservation constraints, capacity constraints, and big-M order/disjunction constraints.
  • Branch and bound solves relaxations for bounds, branches on fractional variables, and prunes impossible or dominated subproblems.
  • Cutting planes and branch-and-cut are useful optional context after branch and bound.
  • Totally unimodular matrices with integral right-hand sides explain why many flow/matching ILPs are polynomial via LP.

Remember: variables model choices, constraints model rules, and the objective says which feasible choice is best.

Module 2

Shortest paths

This module is mostly choosing the right relaxation schedule. Dijkstra finalizes greedily, Bellman-Ford tolerates negative edges, Floyd-Warshall is all-pairs DP, and DAG shortest paths use topological order.

55-70 min

Watch first

Dijkstra explained - FelixTechTips (8:24)

Fallback link

Bellman-Ford in 5 minutes - Michael Sambol (5:09)

Fallback link

Floyd-Warshall - WilliamFiset (15:53)

Fallback link

Say this out loud

What you must be able to say

  • Relaxation checks whether a path through u improves v.
  • Dijkstra requires nonnegative weights and runs in O((n+m) log n) with a binary heap.
  • Bellman-Ford runs in O(nm), handles negative edges, and detects reachable negative cycles.
  • Floyd-Warshall solves all-pairs shortest paths in O(n^3) time and O(n^2) memory.
  • DAG shortest paths process vertices in topological order in O(n+m).

Remember: relaxation always asks one question: does going through this edge improve the current best distance?

Module 3

Network flows, cuts & min-cost flow

Flows are the cleanest KO topic once the residual graph clicks. Everything is about conservation, residual capacity, and certificates: augmenting path for improvement, cut for optimality, negative residual cycle for min-cost improvement.

75-95 min

Watch first

Flow algorithms: max-flow, lower bounds, and min-cost flow

Start with augmenting paths, then use the variants for lower-bound feasibility and cost minimization.

Open current video on YouTube

What they may ask: lower bounds reduce to residual capacities plus balance adjustments; min-cost flow adds edge costs and improves along negative-cost residual cycles.

Say this out loud

What you must be able to say

  • A flow respects capacities and conservation; its value is net flow leaving s.
  • The residual graph has forward capacity u-f and backward capacity f.
  • Ford-Fulkerson repeatedly augments along any residual s-t path by its bottleneck; Edmonds-Karp is the BFS shortest-path variant and is optional implementation detail.
  • Max-flow min-cut says the maximum flow value equals the minimum cut capacity.
  • Lower bounds are removed by f' = f-l and adjusted balances; bipartite matching reduces to unit-capacity max flow.
  • Min-cost flow is optimal iff the residual graph has no negative-cost cycle.
  • Flow formulations cover matching, assignment/transportation, project selection, disjoint paths, image segmentation, and time-expanded scheduling networks.

Remember: the residual graph shows what you can still add on forward edges and undo on backward edges.

Module 4

Knapsack

Knapsack is the place where "pseudo-polynomial" becomes real. The DP looks polynomial in W, but W is a number in the input, not the length of the input encoding.

45-60 min

Watch first

Fractional knapsack - Abdul Bari (15:30)

Fallback link

0/1 Knapsack DP - Abdul Bari (28:24)

Fallback link

Say this out loud

What you must be able to say

  • 0/1 knapsack has binary variables, capacity W, and is weakly NP-hard.
  • Fractional knapsack is solved exactly by ratio greedy.
  • Weight DP runs in O(nW); profit DP runs in O(nP).
  • A 2-approx compares the ratio-greedy prefix with the best single fitting item.
  • FPTAS scales profits, runs profit DP, and gives a (1-epsilon)-approximation.

Remember: pseudo-polynomial means polynomial in the number W or P, not in the number of input bits.

Module 5

Traveling salesman problem

The exam wants the approximation proofs, not just the names. Double-tree and Christofides both start from MST <= OPT, then use triangle inequality to shortcut repeated vertices.

Committee prioritySolid focus

Reports keep returning to approximation versus heuristic answers, so the proof skeleton is higher yield than extra variants.

  • metric TSP assumptions and shortcutting
  • double-tree and Christofides guarantees
  • heuristics, k-OPT, and approximation language
45-60 min

Say this out loud

What you must be able to say

  • TSP is NP-hard; decision TSP is NP-complete.
  • Metric TSP assumes symmetry, nonnegative costs, and triangle inequality.
  • Double-tree: MST, double edges, Euler tour, shortcut; ratio 2.
  • Christofides: MST, odd-degree vertices, minimum perfect matching, Euler tour, shortcut; ratio 3/2.
  • k-OPT is local search: useful heuristic, local optimum, no exact guarantee by default.
  • Metaheuristics may use 2-OPT/3-OPT neighborhoods, but do not claim a ratio unless the method proves one.

Remember: triangle inequality is what makes shortcutting repeated vertices safe.

Module 6

Scheduling

Scheduling is vocabulary-heavy. Learn the notation first, then attach each named official item to its problem class: Bratley for one resource, Horn for preemptive earliest deadline, list scheduling and DP for identical parallel resources, and ILP formulations for project scheduling. LPT and McNaughton are helpful extras, not the first-pass spine.

Say this out loud

What you must be able to say

  • Graham notation is alpha | beta | gamma: machines, restrictions, objective.
  • Job parameters include processing time, release time, hard deadline, due date, weight, completion time, lateness, tardiness, and makespan.
  • Bratley is branch and bound for one-machine release/deadline scheduling.
  • Horn is preemptive earliest-deadline-first for 1|r_j,pmtn|Lmax.
  • List scheduling for P|prec|Cmax has ratio 2 - 1/m; LPT improves independent jobs to 4/3 - 1/(3m).
  • McNaughton is optimal for P|pmtn|Cmax with makespan max(max p_j, sum p_j / m).
  • For fixed m, P_m||Cmax can be solved pseudo-polynomially by DP over machine-load vectors.
  • Project scheduling can use relative-order ILP or time-indexed ILP; temporal constraints connect to shortest paths.

Remember: decode alpha | beta | gamma first; only then pick the matching algorithm or guarantee.

Module 7

Constraint satisfaction & AC3

This is the palate cleanser at the end: variables, domains, constraints, then prune impossible values. AC3 is not magic search; it is local consistency before or during search.

Committee priorityMaximum focus

Multiple 2026 and AI-specialization reports explicitly ask for CSPs, arc consistency, AC3, CSP vs ILP, and Sudoku-style examples.

  • CSP definition: variables, domains, constraints, solution
  • arc consistency, support, revise, and AC3 pseudocode
  • CSP versus ILP and propagation versus search
30-40 min

Watch first

CS50AI Constraint Satisfaction segment (1:44:44)

Fallback link

Stanford CS221 Arc Consistency (14:10)

Excellent short direct source, but Stanford disables embedding on other websites. Use this as the direct-watch fallback if you want the cleanest AC3 explanation.

Watch on YouTube

Watch all 14:10; key moments are 3:10 arc consistency and 8:40 AC3.

Say this out loud

What you must be able to say

  • A CSP has variables, domains, and constraints; a solution assigns values satisfying all constraints.
  • Arc consistency means every value on one side of a binary constraint has support on the other side.
  • AC3 repeatedly revises directed arcs and requeues affected neighboring arcs.
  • If a domain becomes empty, the CSP is inconsistent.
  • AC3 runs in O(e d^3) in the standard bound and is not complete in general.
  • Backtracking searches assignments; forward checking prunes immediate neighbor domains; MAC runs AC3 during search.
  • CSP is feasibility over domains and relations; ILP is linear optimization with integer variables.

Remember: arc consistency deletes unsupported values; it does not guess a full assignment.

Clearly relevant

CSP and AC3

Committee priorityMaximum focus

Directly matches reported CSP/AC3 prompts from the Mgr archive and AI reports.

  • formal CSP definition with Sudoku anchor
  • arc consistency and AC3 queue mechanics
  • CSP and ILP comparison

Context: 2026 Mgr archive and OI Mgr AI reports asked for constraint satisfaction, arc consistency, AC3, CSP vs ILP, Sudoku, pseudocode, and a small example.

Show model answer
  • A CSP has variables X_i, finite domains D_i, and constraints restricting allowed tuples; a solution assigns all variables consistently.
  • Sudoku: variables are cells, domains are digits 1-9, and row/column/box constraints require different digits.
  • Arc (X_i, X_j) is consistent if every value in D_i has some supporting value in D_j satisfying the binary constraint.
  • AC3 queues directed arcs, revises one domain by deleting unsupported values, fails on an empty domain, and requeues incoming neighbor arcs after a deletion.
  • CSP is usually a feasibility/domain-propagation view; ILP uses integer variables, linear constraints, and often an objective, though CSPs can be encoded as ILPs.
Lower confidence

CSP search methods

Context: Bachelor AI/ZUI-style report asked for CSP variables, domains, constraints as relations, and solving with backtracking, forward checking, and arc consistency.

Show model answer
  • Backtracking builds a partial assignment recursively: choose an unassigned variable, try a value, check constraints with assigned neighbors, and undo on failure.
  • Forward checking prunes values from unassigned neighbors that are immediately incompatible with the new assignment.
  • Arc consistency is stronger propagation: every remaining value must have support across each binary constraint.
  • Maintaining arc consistency runs propagation, often AC3, after each assignment; it prunes more but costs more per search node.
  • Propagation alone is not complete for general CSP unless it derives an empty domain or a full assignment; exhaustive backtracking supplies completeness.
Clearly relevant

TSP heuristics and approximation

Reported prioritySolid focus

This prompt appears directly in MSZZ-style reports and is a clean bridge between KO approximation and TAL complexity implications.

  • heuristic vs approximation guarantee
  • metric TSP shortcut proofs
  • 20-OPT and P = NP caveat

Context: MSZZ OI Computer Graphics common-question report asked about TSP heuristics, approximation algorithms, their relationship, an example heuristic, and implications of hypothetical 20-OPT.

Show model answer
  • A heuristic is a practical rule without a required worst-case guarantee, for example nearest neighbor or local improvement.
  • An approximation algorithm has a proven bound: double-tree is a 2-approximation for metric TSP; Christofides is a 3/2-approximation for metric TSP.
  • Both metric guarantees rely on triangle inequality so shortcutting repeated vertices cannot increase the tour length.
  • k-OPT is local search: remove k tour edges, reconnect, accept an improving tour, and stop at a local optimum.
  • If a polynomial-time 20-OPT-like method always found the global optimum for general TSP, then it would solve an NP-hard problem in polynomial time, implying P = NP and NP = coNP; ordinary local search has no such implication.

60-minute panic pass

Minimum viable KO answer

  1. ILP: binary variables model choices; branch and bound uses relaxation bounds; totally unimodular network matrices make some LP relaxations integral.
  2. Shortest paths: Dijkstra nonnegative O((n+m)log n), Bellman-Ford negative edges and cycles O(nm), Floyd-Warshall all pairs O(n^3), DAG topological O(n+m).
  3. Flows: residual graph, Ford-Fulkerson augmenting path, bottleneck, max-flow min-cut, lower-bound/balance reduction, min-cost flow no negative residual cycle.
  4. Knapsack: fractional greedy exact, 0/1 DP O(nW), pseudo-polynomial, 2-approx, FPTAS by profit scaling.
  5. TSP: metric assumption, double-tree 2-approx, Christofides 3/2-approx, k-OPT local search.
  6. Scheduling: alpha|beta|gamma, Bratley, Horn/EDF, list scheduling, dynamic programming for identical machines, project scheduling ILPs. LPT/McNaughton are useful extras if asked.
  7. CSP: variables/domains/constraints, arc consistency, AC3 queue/revise/requeue, O(ed^3), not complete.

Oral rehearsal

Say this three times

Formulations: "Many combinatorial problems can be written as ILPs: binary variables select items, edges, assignments, or orders. Some network-style formulations have integral LP relaxations through total unimodularity, but general ILP is NP-hard, so exact solvers use branch and bound, cuts, and relaxations."

Graph optimization: "Shortest paths are solved by relaxation in different orders; flows are solved by augmenting in the residual graph until a min-cut certificate appears. Minimum-cost flow adds costs and uses the no-negative-residual-cycle optimality condition."

Hard optimization: "For weakly NP-hard knapsack we have pseudo-polynomial DP and an FPTAS. For metric TSP we have approximation algorithms: double-tree gives 2 and Christofides gives 3/2, both relying on triangle inequality."

Scheduling and CSP: "Scheduling is classified by Graham notation and solved by problem-specific exact, greedy, or DP algorithms. CSPs use domain filtering such as AC3, which enforces arc consistency but may still require search."
