PUI · 1 — Planning task: representation → search → guarantees

STRIPS and SAS representation of planning tasks. Satisficing and optimal planning. Heuristic search. Properties of heuristic functions.

1STRIPS task — define FIRST open with this!

say: “A STRIPS task is a factored description of a planning problem: we list facts and actions instead of drawing the state graph — the graph is induced.”

\( \Pi = (F,\, A,\, I,\, G) \)
  • \( F \) = all facts · state \( s \subseteq F \) = facts currently true (rest false — closed world)
  • \( I \) = facts true at start · \( G \) = facts required at end: \( s \models G \iff G \subseteq s \)
  • \( A \) = actions \( a = (\mathit{pre}, \mathit{add}, \mathit{del}) \): needs pre, adds add, deletes del · \( c(a) \ge 0 \)
\( \mathit{pre}(a) \subseteq s \;\Longrightarrow\; s' = \big(s \setminus \mathit{del}(a)\big) \cup \mathit{add}(a) \)
  • no edge if a precondition is missing; unmentioned facts persist (inertia)
  • plan \( \pi = a_1 \ldots a_n \): sequentially applicable from \( I \), ends in \( s \supseteq G \);   \( c(\pi) = \sum_i c(a_i) \)
  • classical assumptions: finite · deterministic · fully observable · static

2SAS⁺ — same world, fewer symbols

say: “SAS⁺ is the same task over multi-valued variables — mutually exclusive STRIPS facts become one variable with a finite domain.”

  • world = variables \( V \); each \( v \) holds one value from a finite domain \( D(v) \) — e.g. \( \mathit{pos}(p) \) with \( D = \{A, B, \mathit{truck}\} \)
  • a state is a total assignment — every variable has exactly one value
  • preconditions / effects are partial assignments — they touch only the variables they care about
  • idea: mutually exclusive facts → one variable
STRIPS (3 facts, mutex)SAS⁺ (1 variable)
\( \mathit{at}(p,A),\; \mathit{at}(p,B),\; \mathit{in}(p,t) \)\( \mathit{pos}(p) \in \{A,\, B,\, \mathit{truck}\} \)
  • same expressiveness; compact + structure for free → projections / PDBs drop variables naturally

3Satisficing vs optimal

say: “Satisficing = any valid plan, fast; optimal = the cheapest plan, plus an implicit proof none cheaper exists.”

4A* — what I'd implement they ask this!

say: “Once the task is fixed, finding a plan = finding a cheapest path in the state graph. A* does that by always expanding the most promising state: \( f = g + h \), cost already paid + cost still estimated.”

keep a queue of states, ordered by f = g + h take the cheapest → goal? done (walk parents back) otherwise: apply every action → queue the successors seen a state before? keep the cheaper path; repeat
  • best known \( g \) per state = duplicate detection; parents rebuild the plan
  • * with consistent \( h \): closed nodes never reopen · only admissible: reopen or lose optimality
  • graph is implicit — successors generated by applying actions, never stored whole

5Heuristic properties

say: “A heuristic estimates remaining cost; its properties set the guarantees — admissibility buys optimality, consistency buys efficiency.”

  • \( h^*(s) \) = true optimal cost-to-goal from \( s \)
  • admissible: \( h(s) \le h^*(s) \;\; \forall s \) — never overestimate
  • consistent: \( h(s) \le c(a) + h(s') \) for every edge, and \( h(s) = 0 \) for goal states
  • goal-aware: \( h(s) = 0 \) when \( s \models G \)  ·  safe: \( h = \infty \) only on dead ends
consistent + goal-aware \( \Rightarrow \) admissible    [induction on plan length]
  • A* + admissible \( h \) ⇒ optimal solution
  • consistent ⇒ \( f \) nondecreasing along paths ⇒ no reopening
  • dominance: \( h_2 \ge h_1 \) (both admissible) ⇒ \( h_2 \) better informed, expands a subset (up to tie-breaking)
  • extremes: \( h \equiv 0 \) → UCS  ·  \( h = h^* \) → expands only the optimal path

⚠ TRAPS

Define \( \Pi \) before any algorithm (reports: examiners stop you otherwise) · missing precondition = no edge, not “fails” · admissible ≠ consistent (consistency is the stronger, per-edge condition) · \( h \equiv 0 \) is admissible — A* degrades to UCS, never breaks · SAS is not “more powerful,” it is more compact.
