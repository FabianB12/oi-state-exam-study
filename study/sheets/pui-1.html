PUI · 1 — Planning task: representation → search → guarantees

STRIPS and SAS representation of planning tasks. Satisficing and optimal planning. Heuristic search. Properties of heuristic functions.

1STRIPS task — define FIRST open with this!

Π = (F, A, I, G)
  • F facts (atoms) · state s ⊆ F (true facts; rest false — closed world)
  • I ⊆ F initial state · G ⊆ F goal: s ⊨ G ⟺ G ⊆ s
  • action a = (pre, add, del), cost c(a) ≥ 0
applicable: pre(a) ⊆ s  →  s′ = (s \ del(a)) ∪ add(a)
  • no edge if precondition missing; unmentioned facts persist (inertia)
  • plan π = a₁…aₙ: sequentially applicable from I, ends in s ⊇ G; c(π) = Σ c(aᵢ)
  • classical assumptions: finite · deterministic · fully observable · static

2SAS⁺ — same world, fewer symbols

  • variables V, each with finite domain D(v); state = total assignment
  • pre / eff = partial assignments
  • idea: mutually exclusive facts → one variable
STRIPS (3 facts, mutex)SAS⁺ (1 variable)
at(pkg,A), at(pkg,B), in(pkg,truck)pos(pkg) ∈ {A, B, truck}
  • same expressiveness; compact + structure for free → projections / PDBs drop variables naturally

4A* — what I'd implement they ask this!

  • OPEN = priority queue by f(s) = g(s) + h(s)
  • store best known g per state (duplicate detection), parent pointers → reconstruct plan
  • CLOSED: with consistent h, closed nodes never reopened
  • with only admissible h: may find better g later → reopen or lose optimality
  • graph is implicit — successors generated by applying actions, never stored whole

5Heuristic properties

  • h*(s) = true optimal cost-to-goal from s
  • admissible: h(s) ≤ h*(s) ∀s — never overestimate
  • consistent: h(s) ≤ c(a) + h(s′) for every edge, h(goal) = 0
  • goal-aware: h(s)=0 when s ⊨ G  ·  safe: h=∞ only on dead ends
consistent (+ goal-aware) ⇒ admissible   [induction on plan length]
  • A* + admissible ⇒ optimal solution
  • consistent ⇒ f nondecreasing along paths ⇒ no reopening
  • dominance: h₂ ≥ h₁ (both adm.) ⇒ h₂ better informed, expands subset (tie-breaks aside)
  • extremes: h=0 → UCS · h=h* → expands only optimal path

⚠ TRAPS

Define Π before any algorithm (reports: examiners stop you otherwise) · missing precondition = no edge, not “fails” · admissible ≠ consistent (consistency is the stronger, per-edge condition) · h=0 is admissible — A* degrades to UCS, never breaks · SAS is not “more powerful,” it is more compact.
