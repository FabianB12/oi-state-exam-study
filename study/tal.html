Module 6

Decidability & undecidability

Where the exam gets philosophical but the answer is mechanical. Decidable means always halts. RE means accepts yes cases eventually. Undecidable proofs usually say: assume a decider, build a contradiction or reduce from a known impossible problem.

Committee prioritySolid focus

Common-core reported questions can be strict here, especially on recursive vs RE language classifications.

  • Decidable/recursive vs RE/recognizable
  • Complement, co-RE, and decidability iff both sides are RE
  • Diagonalization, halting, and reduction proof templates

Clearly relevant

P, NP, coNP, NPC

Committee prioritySolid focus

Reported common-question evidence makes this the TAL drill most likely to punish vague definitions.

  • P/NP/coNP/NPC definitions
  • 3-coloring classification
  • Reduction direction for NP-hardness

Context: OI Mgr Cybersecurity common question asked for class definitions, 3-coloring, polynomial reductions, class relationships, and NP-completeness justification.

Show model answer
  • P: decision problems solvable in deterministic polynomial time.
  • NP: decision problems whose yes-instances have polynomial-size certificates checkable in polynomial time, equivalently accepted by an NTM in polynomial time.
  • coNP: complements of languages in NP; if L is in coNP, no-instances of L have efficiently checkable certificates.
  • 3-coloring is in NP by checking a proposed coloring, and NP-complete via a polynomial reduction from a known NP-complete problem such as SAT/3-SAT.
  • To prove a target problem NP-hard, reduce a known NP-hard problem to the target: A ≤p B means an algorithm for B would solve A.
Clearly relevant

Recursive vs RE

Committee prioritySolid focus

Reported drill with strict terminology: the examiner can quickly separate decider, recognizer, and diagonal-language answers.

  • Recursive vs recursively enumerable languages
  • RE/co-RE complement relationships
  • Diagonal language and complement classification

Context: MSZZ OI Software Engineering common question asked for recursive and recursively enumerable languages, examples, the diagonal language, its complement, and classification.

Show model answer
  • Recursive, or decidable, means there is a Turing machine that halts on every input and accepts exactly the language.
  • Recursively enumerable means there is a recognizer that halts and accepts yes-instances; on no-instances it may reject or loop forever.
  • Every recursive language is RE and co-RE. If a language and its complement are both RE, dovetail the recognizers to decide it.
  • A standard diagonal language says machine M_i does not accept its own encoding/input w_i; diagonalization shows it is not RE.
  • Its complement is RE in the standard setup because a universal machine can simulate M_i(w_i) and accept if it accepts, but it is not recursive.
Clearly relevant

TSP and 20-OPT implications

Reported prioritySolid focus

Not committee-specific, but it has appeared as a common-question prompt and tests whether you can connect algorithms to complexity-class consequences.

  • decision TSP in NP
  • NP-hard exact optimization implication
  • P = NP and NP = coNP consequence

Context: MSZZ OI Computer Graphics common-question report overlapped KO/TAL: TSP heuristics, approximation algorithms, and implications of a hypothetical 20-OPT algorithm for P, NP, NPC, and coNP.

Show model answer
  • Decision TSP is in NP: a tour and bound can be checked in polynomial time.
  • General TSP is NP-hard/NP-complete in decision form, so a polynomial-time exact algorithm for it would imply P = NP.
  • If P = NP, then NP = coNP as well, because P is closed under complement.
  • That would make every NP-complete problem polynomial-time solvable, not just TSP.
  • Caveat: ordinary fixed-neighborhood 20-OPT local search is a heuristic/local optimum method; it only has those implications if it is claimed to always return a global optimum in polynomial time for general TSP.

45-minute panic pass

Minimum viable TAL answer

  1. Correctness: invariant proves partial correctness; variant proves termination; together total correctness.
  2. Machines: DTM has one path, NTM accepts if some branch accepts, recognizer may loop, decider always halts.
  3. Complexity: P deterministic polynomial time; NP polynomial certificate verification; co-NP complements; NPC means in NP and NP-hard.
  4. Reduction: to prove target hard, reduce known hard problem to target. Cook-Levin: SAT is NP-complete via tableau.
  5. Space: PSPACE/NPSPACE use polynomial space; configuration graph reachability; Savitch gives NSPACE(s) subset DSPACE(s^2).
  6. Random: RP false negatives only, co-RP false positives only, ZPP = RP intersection co-RP. BPP is optional context, not the official core.
  7. Undecidable: decidable always halts; RE recognizes yes; A_TM is RE but undecidable; diagonalization also yields non-RE languages.

Oral rehearsal

Say this three times

Complexity classes: "We encode decision problems as languages. P is deterministic polynomial time. NP is nondeterministic polynomial time, equivalently polynomial-time verification with certificates. co-NP contains complements of NP languages."

NP-completeness: "A problem is NP-complete if it is in NP and NP-hard. For hardness, I reduce a known NP-complete problem to the target. Cook-Levin proves SAT is NP-complete by encoding accepting computation histories as Boolean formulas."

Undecidability: "A decidable language has a machine that halts on all inputs. RE only guarantees acceptance of yes-instances. The universal/acceptance language is RE but undecidable, and diagonalization shows that some languages are not even RE. The halting problem is undecidable because a hypothetical halting decider lets us construct a machine that contradicts its own predicted behavior."
