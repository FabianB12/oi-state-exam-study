PUI in Plain English

PUI is about planning actions before acting. The main trick is to estimate distance to the goal using relaxations, abstractions, landmarks, or optimization.

1. State the planning model.States, actions, initial state, goal.
2. Choose the heuristic.Relaxed, abstracted, landmark, LP, or sampled.
3. Say the guarantee.Satisficing, optimal, admissible, probabilistic.

Classical planning

Find an action sequence from initial state to goal.

  • STRIPS: facts, preconditions, add/delete effects.
  • SAS: finite-domain variables.
  • A*, greedy search, and heuristic search are central.

Delete relaxation

Pretend actions never delete facts.

  • hmax: optimistic max cost.
  • hadd: sums subgoal costs, may double count.
  • hff: extracts a relaxed plan.

Abstractions and landmarks

Simplify the problem or identify required facts/actions.

  • Pattern databases solve projected tasks.
  • Merge-and-shrink builds abstract transition systems.
  • Landmarks must be achieved on every plan.

LP heuristics

Use linear constraints to lower-bound plan cost.

  • State-equation heuristic counts action occurrences.
  • Potential heuristics assign values to facts/states.
  • LP gives a numeric estimate for search.

Uncertainty and MCTS

Actions may be nondeterministic or probabilistic.

  • MDP value iteration optimizes expected return.
  • MCTS samples futures instead of expanding all states.
  • UCT balances value and exploration.

How to answer

Always say what information is ignored or approximated.

  • What is the relaxed problem?
  • Is the heuristic admissible?
  • What search uses it?
Exam sentence: Delete-relaxation heuristics estimate goal distance by solving an easier problem where delete effects are ignored, so facts only accumulate.

Three traps

  • Satisficing means "find some plan"; optimal means minimum cost.
  • hadd is usually not admissible because it double-counts shared actions.
  • MCTS is useful when the full search tree is too large to enumerate.
