SMU in Plain English

SMU is about learning from feedback. The main question is always: what feedback do we see, what is the learner trying to minimize, and how much exploration is needed?

1. Identify feedback.Full label, mistake, chosen reward, or delayed reward?
2. Name the goal.Low error, few mistakes, low regret, high return.
3. Give the update.Eliminate, explore, estimate values, or improve policy.

PAC learning

Batch examples from a fixed distribution.

  • Goal: low true error with high probability.
  • Needs sample complexity in epsilon and delta.
  • VC dimension controls learnability.

Online learning

Predict one example at a time.

  • Goal: few mistakes or low regret.
  • Can be adversarial sequence.
  • Conjunction/disjunction learners eliminate inconsistent literals.

Bandits

You only observe reward for the arm you pulled.

  • Explore/exploit tension is central.
  • Regret compares against best fixed arm.
  • UCB = average reward plus uncertainty bonus.

Reinforcement learning

Actions change future states and rewards.

  • Value iteration assumes known model.
  • TD/Q-learning learn from sampled transitions.
  • Policy search optimizes policy directly.

Key contrast

Less feedback makes learning harder.

  • PAC: label for sampled example.
  • Bandit: only chosen reward.
  • RL: chosen reward plus next state, delayed consequences.

How to answer

Always say the feedback model first.

  • What is observed?
  • What is optimized?
  • What algorithm handles exploration?
Exam sentence: In bandits the learner does not see rewards of unchosen arms, so algorithms like UCB add an uncertainty bonus to try promising but under-sampled actions.

Three traps

  • PAC assumes i.i.d. data; online learning may be adversarial.
  • Bandit regret is not the same as supervised classification error.
  • Q-learning estimates action values, not just state values.
