PAL in Plain English

PAL is not one big idea. It is five small toolboxes. Learn what each toolbox is for, the main names, and one sentence you can say in the exam.

1. Recognize the topic.Graph, generation, tree, or automaton?
2. Say the landmark.Condition, algorithm name, or formula.
3. Finish with complexity.Most PAL answers end with O(...).
1

Graph basics

How to store a graph and talk about its size.

Know this

  • n = vertices, m = edges.
  • Adjacency list: good for sparse graphs, O(n + m) memory.
  • Adjacency matrix: O(1) edge lookup, but O(n^2) memory.

Watch out

  • Distance matrix usually means shortest-path distances.
  • Laplacian/incidence matrices are the algebraic view.
Exam sentence: For sparse graphs I would use adjacency lists because they take O(n + m) memory; matrices are useful when I need O(1) edge queries.
2

Graph algorithms

MST, components, Euler trails, and isomorphism.

Core algorithms

  • Prim/Kruskal/Boruvka build minimum spanning trees.
  • Union-find tracks components for Kruskal.
  • Kosaraju/Tarjan find SCCs in O(n + m).

Key conditions

  • MST: cut property and cycle property.
  • Euler undirected: all even, or exactly two odd.
  • Euler directed: in-degree/out-degree balance.
Exam sentence: Kruskal sorts edges and adds an edge only if union-find says it connects two different components; sorting gives O(m log n).
3

Generation and numbers

Listing objects, finding primes, and simple pseudorandomness.

Counts

  • Subsets: 2^n.
  • k-subsets: C(n, k).
  • Permutations: n!.

Names to remember

  • Gray code: consecutive bit strings differ in one bit.
  • Sieve of Eratosthenes: all primes up to N.
  • LCG: X' = (aX + c) mod m.
Exam sentence: The sieve marks multiples starting from p^2 and finds all primes up to N in O(N log log N) time.
